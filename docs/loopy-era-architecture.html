Loopy Era

Claude Code Autonomous Self-Improving Architecture
"The human sets direction. The system builds itself."

💡 Core Concept

🔄 Self-Improving Loop
Every mistake teaches the system. Fix commits are detected → analyzed → new rules are added to project scaffolds → same mistake never happens again. The scaffold evolves autonomously.
🤖 Zero Human Intervention
30 hooks across 11 events handle everything: validation errors auto-fix via Opus agent hooks, test failures trigger retries with strategy rotation, and HARD blocks send Telegram alerts only when truly stuck.
🧠 Cross-Session Memory
Memory Bank indexes 50K+ conversations with semantic search. Facts (decisions, preferences, patterns) persist across sessions. Each session starts with context injection from past work.

🏗️ Architecture Layers

👤 User Layer
Direction setting, feedback, quality information input. Telegram notifications for HARD blocks. The human is the "taste maker" — not the worker.
🧠 Claude Layer
Opus 4.6 main thread + orchestrators (manager/team). Skills provide domain knowledge. Agents handle specialized tasks. bypassPermissions mode — no approval prompts.
⚡ Hook Layer
30 hooks enforce quality gates. HARD gates block bad commits. AGENT hooks (Opus) auto-fix errors. SOFT hooks inject guidance via additionalContext.
💾 Memory Layer
Memory Bank (2,600+ facts, semantic search), session state persistence, project scaffolds (NEVER DO rules), error logs (tool-errors.jsonl), self-improve pending queue.
🔧 Infra Layer
Git (atomic commits), npm/tsc/vitest (build/test), SQLite + sqlite-vec (vector DB), Telegram bot (notifications), MCP servers (Supabase, Figma, browser).

🚨 3-Stage Error Escalation

When code has errors, the system escalates automatically — no human needed until Stage 3.

Stage 1 — Soft
additionalContext
Error message + "fix this now" injected into Claude's context. Claude usually fixes it immediately.
1/3 attempts
Stage 2 — Agent
Opus Agent Hook
Independent Opus agent spawned by hook. Reads file, analyzes error, fixes it directly. Runs parallel to main thread.
2/3 attempts
Stage 3 — Hard Block
exit 2 + Telegram
Tool execution blocked. Telegram alert sent to user. "Change your approach" message. Counter resets.
blocked

🔄 Self-Improvement Loop

Session N: Development work ↓ fix: commit detected → Stop hook (self-improve-trigger.sh) → saves ~/.claude/pending/self-improve-{project}.json ↓ Session N+1: User sends any prompt → UserPromptSubmit hook (self-improve-check.sh) → pending file found → "run /self-improve first" auto-injected ↓ Claude runs /self-improve → analyzes fix diff → adds new NEVER DO rule to scaffold → deletes pending file ↓ Session N+2: Scaffold is stronger. Same mistake cannot happen again.

🧠 Memory System

📚 Memory Bank Plugin
50K+ conversations indexed with semantic search (all-MiniLM-L6-v2, 384-dim vectors).
2,600+ facts extracted by Haiku — decisions, preferences, patterns, knowledge, constraints.
Ontology — 20 domains, auto-classified, with INFLUENCES/SUPPORTS/CONTRADICTS relations.
3D Knowledge Graph — visual explorer with 4 layers (prompt → user → project → ontology).
📋 Project Scaffolds
Per-project SKILL.md files with NEVER DO rules extracted from actual fix commits.
scaffold-violation-check.sh hook enforces rules at edit time.
self-improve adds new rules automatically when patterns are detected.
Created by /init-project — analyzes real codebase, not generic templates.
🔄 Session Continuity
session-state-save — captures uncommitted changes + last commit at session end.
session-resume-check — restores state at next session start.
context-snapshot/restore — preserves work context across compact cycles.
inject-context — top facts + recent session summary injected before each prompt.

👤 Human's Role in Loopy Era

✅ What Humans Do
• Set direction and goals
• Provide quality feedback
• Feed quality information (articles, repos, ideas)
• Respond to Telegram HARD-block alerts
• Approve/reject self-improve proposals
❌ What Humans Don't Do
• Write code
• Create skills manually (skills create skills)
• Fix bugs (3-stage escalation handles it)
• Run tests (hooks auto-validate)
• Approve tool execution (bypassPermissions)
